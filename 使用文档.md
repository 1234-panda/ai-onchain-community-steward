# AI 链上身份社区管家 — 使用文档

## 项目是什么

一个 Discord-first 的 AI + 区块链社区治理工具。成员通过 MetaMask 绑定钱包建立链上身份，Bot 实时监控 Discord 消息并检测广告/诈骗/FUD 风险，管理员在 Discord 内完成全部审核操作后批量写入 Sepolia 测试网。链上信誉跨服务器可查，不可转让的社区通行证 NFT 作为防女巫攻击的身份凭证。

│  AI Agent: DeepSeek/OpenAI → 风险建议                 │
│  规则引擎: 关键词检测 + 评分模型                       │
│  存储: data/community.json（本地 MVP）                │
└────────────────────┬─────────────────────────────────┘
                     │
┌────────────────────▼─────────────────────────────────┐

│  CommunityReputation: recordEvent / batchRecordEvents │

## 第一次启动（一步步）

### 第一步：安装依赖

```bash
git clone <仓库地址>
cd project
npm install
```

### 第二步：配置环境变量

```bash
cp .env.example .env.local
```

编辑 `.env.local`，填入以下内容：

```env
# ── Discord ──
DISCORD_BOT_TOKEN=            # Discord Developer Portal → Bot → Token
DISCORD_CLIENT_ID=            # Discord Developer Portal → OAuth2 → Client ID
DISCORD_GUILD_ID=             # 你的服务器 ID（开启开发者模式后右键服务器图标 → 复制 ID）
DISCORD_ADMIN_CHANNEL_ID=     # 管理员通知频道 ID（右键频道 → 复制 ID）
ADMIN_DASHBOARD_PASSWORD=     # 管理员 Dashboard 密码（自己设）

# ── Sepolia ──
CHAIN_ID=11155111
EVM_RPC_URL=https://sepolia.infura.io/v3/你的KEY
EXPLORER_BASE_URL=https://sepolia.etherscan.io
SERVICE_WALLET_PRIVATE_KEY=   # 你有 Sepolia ETH 的测试钱包私钥
TRUSTED_GUILD_IDS=            # 可信服务器白名单，填你的服务器 ID
CHAIN_WRITE_MODE=manual       # 推荐 manual，攒着批量写链省 Gas

# ── AI（可选）──
LLM_API_KEY=                  # DeepSeek 或 OpenAI API Key
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat

# ── 其他 ──
NEXT_PUBLIC_APP_URL=http://localhost:3000
DEMO_MODE=true
```

### 第三步：开启 Discord Message Content Intent

打开 [discord.com/developers/applications](https://discord.com/developers/applications) →
你的应用 → Bot → Privileged Gateway Intents → 打开 **Message Content Intent** → 保存。

不打开 Bot 读不到用户消息内容，规则引擎和 AI 全部失效。

### 第四步：部署合约到 Sepolia

```bash
npm run deploy:sepolia        # 信誉合约
npm run deploy:pass:sepolia   # 社区通行证合约
```

部署成功后会输出合约地址，分别填入 `.env.local`：

```env
REPUTATION_CONTRACT_ADDRESS=0x...
VIP_NFT_ADDRESS=0x...
```

### 第五步：配置检查

```bash
npm run doctor
```

确保关键项都是 `OK`，RPC 实际 chainId 是 `11155111`，服务钱包有 Sepolia ETH。

### 第六步：注册 Discord 命令

```bash
npm run register:commands
```

### 第七步：启动服务（分两个终端）

```bash
# 终端 A：先启动 Next.js
npm run dev
# 等到出现 ✓ Ready in XXs

# 终端 B：再启动 Bot
npm run bot
# 等到出现 Logged in as AI 社区管家#xxxx
```

### 第八步：验证

- 浏览器打开 `http://localhost:3000/setup` → 确认所有配置项 OK
- Discord 里输入 `/help` → Bot 应该回复帮助信息

| 命令 | 功能 |
|------|------|
| `/config admin-channel #频道` | 设置当前服务器的管理员操作台频道 |

## 日常使用流程

### 用户绑定钱包

```
1. 用户在 Discord 输入 /bind
2. Bot 返回绑定链接 → 用户浏览器打开
3. 页面提示连接 MetaMask → 签名（不花 Gas，纯粹证明身份）
4. 绑定成功后页面显示当前身份等级和信誉分
5. 绑定前未绑钱包时积累的违规记录（awaiting_wallet）自动补写链上
```

### 消息实时监控（全自动）

```
1. 用户在 Discord 发消息
2. Bot 实时扫描内容，命中关键词 → 创建待处理事件
3. 通知推送到管理员频道，带交互按钮：[Confirm] [Dismiss]
4. AI 分析最近消息生成风险建议（源频道限流 2 条/小时）
```

### 管理员确认事件

```
1. #bot-通知 频道出现风险卡片
2. 管理员点 [Confirm] → 模态框弹出
   → 系统自动预填原因和扣分（管理员可修改，范围限制内）
   → 提交后 chainStatus 变为 pending
3. 管理员点 [Dismiss] → 事件直接消除，不扣分不写链
4. 攒够若干条 pending 后 → 输入 /write-batch limit:10
   → 所有 pending 打包成一笔 Sepolia 交易
   → 多条记录共享同一个 txHash
```

### 主动奖励

```
管理员看到 @熊猫 热心帮新人答疑
→ /positive-contribution @熊猫 热心答疑
→ 创建 POSITIVE_CONTRIBUTION +15 待确认事件
→ Confirm → pending → /write-batch 上链
→ 熊猫的信任分上升
```

### 用户申诉

```
被扣分用户输入 /appeal 我昨天的发言正常，被误判了
→ 管理员频道出现申诉卡片：[Confirm] [Dismiss]
→ 管理员复核后：
   Confirm = 接受申诉 +20，Dismiss = 驳回
```

### 发通行证

```
管理员 /risk user:@熊猫 → 查看完整画像
→ 确认可信 → /issue-pass @熊猫
→ 服务钱包调用 MemberPassNFT.mint() → Sepolia 上链
→ 用户持有通行证 → 信任分 +8，女巫风险降低
```

## 评分机制

### 自动计算部分

| 项目 | 分数 |
|------|------|
| 基础分 | 60 |
| 持有 Token ≥ 10 个 | +35 |
| 持有社区通行证 NFT | +8 |
| 未绑定钱包 | -20 |
| 链上全局信誉贡献 | -40 ~ +20 |
| 女巫风险扣分 | 0 ~ -60 |

### 管理员手动裁决（范围限制）

| 事件类型 | 范围 | 默认值 |
|---------|------|--------|
| WARNING / FUD | 0 ~ -20 | -5 |
| SPAM | -10 ~ -50 | -30 |
| SCAM | -30 ~ -100 | -50 |
| 积极贡献 (Positive) | +5 ~ +20 | +15 |
| 申诉通过 (Appeal) | +5 ~ +30 | +20 |

管理员在范围内自由裁量，超出范围会被拒绝。系统预填默认值，管理员可改但不可离谱。

## 女巫防护

系统不做 KYC，只根据可观察信号调整审查强度：

| 信号 | 风险分 |
|------|--------|
| 未绑定钱包 | +25 |
| 钱包绑定不到 24 小时 | +10 |
| 无通行证且无 Token 持仓 | +10 |
| 链上无信誉记录 | +10 |
| 同一钱包绑定多个 Discord 号（同服务器） | +30 |
| 同一 Discord 号换绑多个钱包 | +15 ~ +25 |
| 近期换绑钱包（7 天内） | +10 |
| 历史钱包有负面记录 | +25 |

这些信号只表示"需要更严格审查"，不是自动定罪。

## 可信 Guild 白名单

`.env.local` 中的 `TRUSTED_GUILD_IDS` 控制哪些服务器能写全局 Sepolia 信誉和发通行证：

```env
# 只有你的服务器能写链
TRUSTED_GUILD_IDS=1485817642219733143

# 你的 + 朋友的服务器
TRUSTED_GUILD_IDS=1485817642219733143,9999999999999999999
```

不在白名单的服务器只能做本地治理（`chainStatus: local_only`），不能碰全局链上信誉。

## 链上存什么

```
链上（Sepolia 合约）        链下（data/community.json）
┌──────────────────┐       ┌──────────────────────────┐
│ wallet: 0x0C46.. │       │ discordId: "148580..."    │
│ eventHash: 0xab..│ ←→   │ walletAddress: 0x0C46...  │
│ eventType: 1     │       │ eventType: SPAM          │
│ scoreDelta: -30  │       │ scoreDelta: -30          │
└──────────────────┘       │ reason: "疑似广告"        │
                           │ messageSummary: "稳赚..."  │
                           │ eventHash: 0xab...        │
                           │ txHash: 0x...             │
                           └──────────────────────────┘

验证方式：任何人拿链下原始数据重新算 keccak256，
结果和链上 eventHash 一致则证明记录未被篡改。
```

---

## 隐私边界

- 链上**不存** Discord ID、用户名、聊天原文
- 链上只存 eventHash（keccak256 指纹）、事件类型、分数变化
- 女巫风险只是信号提示，不自动封禁
- AI 只生成建议，最终治理动作必须由管理员确认

## 常见问题

**Q: Bot 在 Discord 里没反应？**
A: ① `npm run dev` 和 `npm run bot` 是否都在跑 ② Message Content Intent 是否打开 ③ `npm run register:commands` 是否执行过 ④ Bot 是否已被邀请进服务器。

**Q: 钱包绑定失败？**
A: ① 浏览器是否装了 MetaMask ② 签名是否在 10 分钟内完成 ③ 是否重复提交了同一个 challenge。

**Q: Gas 消耗太大？**
A: 推荐 `CHAIN_WRITE_MODE=manual`，管理员攒够 pending 后用 `/write-batch limit:10` 批量写链，Gas 省 ~80%。Sepolia 测试币免费。

**Q: 不填 LLM_API_KEY 能用吗？**
A: 能。系统自动降级为规则兜底——统计风险词命中数生成摘要建议。

**Q: 数据存在哪？**
A: 本地 `data/community.json`。该文件已加入 `.gitignore`，不会上传到 GitHub。

**Q: 切回本地 Hardhat？**
A: 改 `.env.local`：`CHAIN_ID=31337`，`EVM_RPC_URL=http://127.0.0.1:8545`，`EXPLORER_BASE_URL=`，然后 `npm run hardhat:node` + `npm run deploy:local`。

- ⚠️ `.env.local` 不要提交到 Git（已在 `.gitignore` 中）
- Discord Bot Token 暴露后必须去 Developer Portal 重置
- LLM API Key 暴露后必须去对应平台重置
